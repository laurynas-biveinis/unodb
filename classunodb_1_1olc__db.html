Detailed Description

template<typename Key>
class unodb::olc_db< Key >

A thread-safe Adaptive Radix Tree that is synchronized using optimistic lock coupling.

At any time, at most two directly-related tree nodes can be write-locked by the insert algorithm and three by the delete algorithm. The lock used is optimistic lock (see optimistic_lock.hpp), where only writers lock and readers access nodes optimistically with node version checks. For deleted node reclamation, Quiescent State-Based Reclamation is used.

Scan a half-open key range, applying the caller's lambda to each visited leaf.

The scan will proceed in lexicographic order iff from_key is less than to_key and in reverse lexicographic order iff to_key is less than from_key. When from_key < to_key, the scan will visit all index entries in the half-open range [from_key,to_key) in forward order. Otherwise the scan will visit all index entries in the half-open range (from_key,to_key] in reverse order.

Parameters
from_keyis an inclusive bound for the starting point of the scan.
to_keyis an exclusive bound for the ending point of the scan.
fnA function f(unodb::visitor<unodb::olc_db::iterator>&) returning bool. The traversal will halt if the function returns true.
